Maven Jetty 插件是一个流行的工具，允许开发者直接在他们的 Maven 项目中使用嵌入式 Jetty 服务器运行 Java 网络应用程序。

Jetty 插件启动一个嵌入式 Jetty 服务器，该服务器监视您应用程序的文件，包括 Java 类和资源，以检测更改。当它检测到更新时，它会自动重新部署应用程序，从而加快开发速度，消除手动构建和部署步骤。

:::tip 前端更改
`src/main/frontend` 下的更改由 [frontend watch](/docs/configuration/deploy-reload/frontend-watch) 处理，它重建这些更改并在服务器旁边刷新浏览器。
:::

## Jetty 配置 {#jetty-configurations}

以下是一些重要配置，用于微调插件的热部署和服务器交互设置：

| 属性                          | 描述                                                                                                                                                                           | 默认值        |
|-------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------|
| **`scan`**                    | 配置 Jetty 服务器每隔多长时间扫描 **`pom.xml`** 中的文件更改。骨架项目将此设置为 `2` 秒。增加此间隔可以减少 CPU 负载，但可能会延迟更改在应用中的反映。 | `1`            |

## webforJ 配置 {#webforj-configurations}

| 属性                          | 描述                                                                                                                                                                           | 默认值        |
|-------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------|
| **`webforj.reloadOnServerError`** | 使用热重新部署时，整个 WAR 文件会被替换。如果客户端在服务器重启时发送请求，会发生错误。该设置允许客户端尝试页面重新加载，假设服务器很快会恢复在线。仅适用于开发环境，并且仅处理与热重新部署相关的特定错误。 | `on`           |
| **`webforj.clientHeartbeatRate`** | 设置客户端轮询以查询服务器可用性的时间间隔。这保持了客户端与服务器之间的通信开放。在开发中，使用更短的间隔以便更快地检测错误。在生产环境中，将其设置为至少 50 秒以避免过多请求。 | `50s`          |

## 使用注意事项 {#usage-considerations}

尽管 Jetty 插件在开发中非常有效，但它有一些潜在的限制：

- **内存和 CPU 使用**：频繁的文件扫描（在 `pom.xml` 中由较低的 `scan` 值设置）会增加资源消耗，尤其是在大型项目中。增加间隔可能会减少负载，但也会减慢重新部署速度。

- **有限的生产使用**：Jetty 插件旨在用于开发，而不是生产环境。它缺乏生产所需的性能优化和安全配置，更适合本地测试。

- **会话管理**：在热重新部署期间，用户会话可能不会被保留，尤其是在代码发生重大结构变化时。这可能会干扰涉及用户会话数据的测试，要求手动管理会话或为开发配置解决方法。
