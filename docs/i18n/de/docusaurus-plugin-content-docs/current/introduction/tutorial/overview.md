Dieses schrittweise Tutorial führt Sie durch den Prozess der Erstellung einer Kundenverwaltungs-App mit webforJ und Spring Boot. Es zeigt Ihnen, wie Sie eine moderne, benutzerfreundliche Oberfläche zum Anzeigen, Hinzufügen und Bearbeiten von Kundendaten erstellen.

Jeder Schritt führt zu neuen Konzepten und resultiert in einer ausführbaren Spring Boot-App (JAR). Sie können Ihre App lokal mit Maven starten und in einem Webbrowser damit interagieren. Mit diesem Setup erhalten Sie einen schnellen Entwicklungszyklus mit Frontend-Bündelung und einem produktionsbereiten Bereitstellungsmodell, das den eingebetteten Server von Spring Boot nutzt.

Es sind keine Vorkenntnisse in Spring Boot oder webforJ erforderlich, aber Sie sollten ein grundlegendes Verständnis von Java und Maven haben, um das Beste aus diesem Tutorial herauszuholen. Dieses Tutorial behandelt Spring-Konzepte, wenn sie auftauchen, aber diejenigen, die ein vertieftes Verständnis von Spring wünschen, können die [Hauptdokumentation von Spring](https://spring.io/learn) und die Dokumentation von Spring über [Spring Boot](https://docs.spring.io/spring-boot/index.html) besuchen.

## Tutorielles Konzepte {#tutorial-concepts}

Der erste Teil des Tutorials ist der [Projekteinrichtung](/docs/introduction/tutorial/project-setup) gewidmet, um Ihre Spring Boot + webforJ-Umgebung vorzubereiten. Anschließend führen die folgenden Schritte neue Funktionen ein und entwickeln Ihr Projekt weiter. Indem Sie mitmachen, werden Sie ein klares Verständnis dafür gewinnen, wie sich eine App entwickelt, während Sie Funktionen implementieren.

## Voraussetzungen {#prerequisites}

Sie sollten die folgenden Werkzeuge/Ressourcen auf Ihrem Entwicklungsrechner haben:

- Java 21 oder 25
- Maven
- Eine Java-IDE
- Git (empfohlen, aber nicht erforderlich)

:::info Voraussetzungen von webforJ
Überprüfen Sie den [Artikel zu den Voraussetzungen](/docs/introduction/prerequisites) für einen detaillierteren Überblick über die erforderlichen Werkzeuge für Ihre Entwicklungsumgebung.
:::
