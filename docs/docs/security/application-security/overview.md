Running your app in Java on the server (see [Client and server](/docs/architecture/client-server)) contains whole classes of web attacks before you write a line of code. It doesn't remove your responsibilities, though. These pages explain where the line falls, what webforJ handles for you, where you stay in charge, and how to keep a production deployment locked down.

For controlling who can reach each view, see the rest of the [Security](/docs/security/overview) section on authentication and route-level authorization.
