title: Local BLS License

description: Configure a webforJ project to use a locally installed BLS certificate and client configuration.

A local BLS lets a webforJ app request a license from a service running on your development machine or internal network. This setup is useful when you have a serial number and authorization number and want the project to use the generated local license files instead of the default license configuration.

The usual startforJ project layout includes two license files under `src/main/resources`:

```text
src/main/resources/blsclient.conf
src/main/resources/certificate.bls
```

To use a local BLS, replace those project files with the matching files generated by your local BLS installation.

## Prerequisites {#prerequisites}

Before you start, make sure you have:

- Java 21 or Java 25 available to run the BLS 26 installer.
- A serial number and authorization number.
- A webforJ project with a `src/main/resources` directory.
- Access to the machine where BLS will run.

## 1. Download the BLS installer {#1-download-bls}

Go to the [downloads page](https://basis.cloud/bbj-downloads/), select `BLS` as the product, fill in the required contact information, accept the terms, and download the installer JAR.

![Download form with BLS selected as the product](/img/configuration/local-bls-license/download-bls.png#rounded-border)

*Download form with `BLS` selected as the product.*

:::note
The BLS 26 download page indicates that Java 21 or Java 25 is required.
:::

## 2. Configure the local BLS {#2-configure-bls}

1. Run the downloaded installer:

   ```bash
   java -jar <downloaded-bls-installer>.jar
   ```

2. Follow the installer prompts and fill in the required details.

   ![License Registration Wizard with Retrieve a license selected](/img/configuration/local-bls-license/retrieve-license.png#rounded-border)

   *License Registration Wizard with `Retrieve a license` selected.*

3. When the License Registration Wizard opens, keep the default `Retrieve a license` option selected.
4. Enter your contact information, serial number, and authorization number.
5. Choose `Register and install a license automatically`.
6. Finish the configuration and start BLS.

## 3. Copy the generated license files {#3-copy-the-generated-license-files}

In your webforJ project, replace these files:

```text
src/main/resources/blsclient.conf
src/main/resources/certificate.bls
```

![startforJ project resources containing blsclient.conf and certificate.bls](/img/configuration/local-bls-license/project-resources.png#rounded-border)

*startforJ project resources containing `blsclient.conf` and `certificate.bls`.*

with the files generated in the local BLS installation folder:

```text
C:\bbx\bls\cfg\blsclient.conf
C:\bbx\bls\cfg\certificate.bls
```

![BLS cfg folder containing the generated client configuration and certificate](/img/configuration/local-bls-license/bls-cfg-folder.png#rounded-border)

*BLS installation `cfg` folder containing the generated client configuration and certificate.*

Copy only `blsclient.conf` and `certificate.bls`. Don't copy `.sample`, `.bak`, or `license.bls` files into the webforJ project.

:::tip
If your license files live outside the default webforJ configuration directory, you can configure the license directory with [`webforj.license.cfg`](./properties#configuration-options).
:::
